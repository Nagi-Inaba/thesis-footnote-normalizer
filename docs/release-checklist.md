# 公開前チェック

この文書は、リポジトリを更新・公開する保守者向けです。
監査ツールを利用するだけの場合は実施する必要がありません。

## 1. 公開する差分を確定する

```powershell
git status --short --branch
git diff --name-status
git diff --check
```

意図したコード、テスト、文書だけが差分に含まれていることを確認します。
作業用ファイルを除外するために、`.gitignore`だけへ依存しないでください。

## 2. 原稿と生成物を除外する

次のファイルをコミットしません。

- 実際の論文・参考文献を含む`.doc`、`.docx`、`.docm`、PDF
- `input/`、`work/`、`output/`内のファイル
- 実論文から生成した`footnotes.csv`、`citations.csv`、`issues.csv`、`summary.json`、`report.md`
- 個人名、メールアドレス、未公表の引用、顧客情報を含むデータ

テストで使うDOCXは、架空のデータだけから実行時に生成します。

## 3. 秘密情報と環境依存情報を確認する

追跡済みファイルと、Gitが除外していない未追跡ファイルに次の情報がないことを確認します。

- APIキー、アクセストークン、秘密鍵
- ユーザープロファイルや作業ディレクトリの絶対パス
- `file://` URL、UNCパス、特定端末だけで有効なパス
- 実原稿や監査結果へのリンク

公開文書の例では、リポジトリ相対パス、環境変数、プレースホルダーを使用します。

## 4. リポジトリ全体を検証する

```powershell
.\scripts\Test-Repository.ps1
```

少なくとも次の結果を確認します。

- 監査テストが成功する
- 入力DOCXのSHA-256が実行前後で一致する
- 同じ入力から決定的な出力を得られる
- Codex・Claude Codeアダプターのインストール試験が成功する
- 追跡済みファイルと、Gitが除外していない未追跡ファイルを対象にした公開ツリー検査が成功する

## 5. コミットと公開を確認する

コミット前に作者名を確認し、メールアドレスが`@users.noreply.github.com`で終わることを確認します。

```powershell
git config --get user.name
git config --get user.email
```

コミット後、プッシュ前に、実際のコミットのAuthorとCommitter情報を確認します。
両方のメールアドレスが`@users.noreply.github.com`で終わっていなければ、プッシュしません。

```powershell
git show -s --format=fuller HEAD
```

プッシュ後は、GitHub上の`main`がローカルHEADと一致し、GitHub Actionsの`validate`が成功したことを確認します。
脆弱性の非公開報告窓口とライセンス表示も維持してください。

## 公開可の条件

次の条件をすべて満たした場合に公開可とします。

- 原稿、実データ、秘密情報、ローカル絶対パスが公開対象にない
- 全リポジトリテストが成功している
- 実際のコミットのAuthorとCommitterメールがGitHubのnoreplyアドレスである
- 入力を変更しない監査境界と、人による最終承認の説明が維持されている
- リモートHEADとCI結果を確認している
